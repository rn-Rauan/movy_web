# Instruções para Lovable

## Padrões de Código Obrigatórios

### Tipos TypeScript
- **Nunca use `any`** — use `unknown` para valores dinâmicos
- Imports de tipos do `@/lib/types.ts`: `AuthUser`, `TripInstance`, `Booking`, etc.
- Tipos genéricos: `api<T = unknown>(...)` em vez de `T = any`

### Tratamento de Erros
- Use `catch (err)` sem tipo
- Acesse com: `err instanceof Error ? err.message : "Erro desconhecido"`
- Evite: `catch (err: any)` ou `catch (err as Error)`

### Casts TypeScript
- Casts apenas quando absolutamente necessário
- Exemplo correto: `v as "ONE_WAY" | "ROUND_TRIP"` (tipo literal específico)
- Evite: `as any`, `as Object`, `as unknown`

### ESLint
- Respeite `eslint.config.js` — não ignore warnings
- Use `// eslint-disable-next-line` apenas com justificativa
- Rodar `npm run lint` após qualquer alteração

## Estrutura Preservada
- Rotas em `src/routes/` — naming: `file.route.tsx` para `/file/route`
- Componentes UI em `src/components/ui/` — do shadcn/ui
- Types em `src/lib/types.ts` — exportar novos tipos aqui
- Documentação em `/docs/` — NÃO alterar
